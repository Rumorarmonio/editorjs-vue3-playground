# Назначение

Этот skill используется, когда в ходе одной рабочей сессии Codex несколько раз предлагал сообщения для коммита, а пользователю нужно быстро увидеть, что было сделано, какие баги были исправлены и какое итоговое commit-сообщение лучше использовать.

# Когда использовать

Используй этот skill, если:

- пользователь просит собрать commit-сообщения из текущей сессии;
- пользователь хочет кратко увидеть, что было реализовано и исправлено;
- в сессии уже были ответы Codex с блоками вида `RU:` и `EN:`;
- нужно выбрать финальное сообщение для коммита;
- нужно отличить основную реализованную фичу от последующих багфиксов и правок после review.

# Когда не использовать

Не используй этот skill, если:

- пользователь просит написать новое commit-сообщение с нуля только по текущему diff;
- пользователь просит выполнить сам commit;
- в текущей сессии не было предложенных commit-сообщений;
- нужно анализировать git history, а не текущую переписку;
- нужно менять файлы проекта.

# Правила работы

1. Не изменяй файлы проекта.
2. Не запускай команды, которые меняют рабочее дерево.
3. Просмотри текущую сессию и найди все предложенные commit-сообщения в формате:
    - `RU: ...`
    - `EN: ...`
4. Собери только сообщения, которые относятся к фактически выполненным изменениям.
5. Игнорируй сообщения, которые:
    - были черновыми;
    - были заменены более точными;
    - относятся к отменённым изменениям;
    - явно не соответствуют итоговому состоянию задачи.
6. Раздели найденные сообщения на группы:
    - основная фича;
    - исправления багов;
    - правки после review;
    - технические/вспомогательные изменения, если они были.
7. Если первое commit-сообщение лучше всего отражает основную цель задачи, отметь его как рекомендуемое итоговое сообщение.
8. Если в ходе правок смысл задачи изменился, предложи обновлённое итоговое сообщение.
9. Не придумывай изменения, которых не было в сессии.
10. Если информации недостаточно, явно укажи, что часть выводов основана только на сообщениях из текущей сессии, а не на diff.

# Формат ответа

Ответь кратко и структурированно:

## Найденные сообщения

Список всех релевантных сообщений:

- RU: ...
  EN: ...

## Что было сделано

Краткий список по смыслу:

- реализовано: ...
- исправлено: ...
- доработано после review: ...

## Рекомендуемое итоговое commit-сообщение

- RU: ...
- EN: ...
